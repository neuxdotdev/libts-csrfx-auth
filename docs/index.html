    CSRF-X Auth Library - v0.1.1

    libts-csrfx-auth

    Minimal TypeScript auth handler for web sessions, CSRF tokens, and cookie persistence — inspired by Rust's cekunit-client.

    Installation 

    # Using Bun (recommended)
    bun add libts-csrfx-auth

    # Using npm
    npm install libts-csrfx-auth

    Prerequisites: Node.js >=18 or Bun >=1.0.

    Quick Start

    Set up your environment variables (.env file or system environment): 

    BASE_URL=https://example.com
LOGIN_PATH=login
LOGOUT_PATH=logout
USER_EMAIL=you@example.com
USER_PASSWORD=yourpassword

    Then use the library:

    import { AuthClient, LogoutClient } from 'libts-csrfx-auth'

    const auth = new AuthClient()
    const session = await auth.login()
    console.log(`Logged in! CSRF token: ${session.csrfToken}`)

    // Later, to log out
    const logout = new LogoutClient()
    await logout.logout()

    Checking cached session before login 

    if (await auth.hasValidSession()) {
    	const session = await auth.getCachedSession()
    	console.log('Reusing session from cache')
    } else {
    	await auth.login()
    }

    Library API

    AuthClient

    The main client for CSRF‑protected login flows.

    class AuthClient {
    	constructor(options?: LoginOptions)
    	login(): Promise<SessionData>
    	getCachedSession(): Promise<SessionData | null>
    	hasValidSession(maxAgeMs?: number): Promise<boolean>
    	clearSession(): Promise<void>
    	readonly cacheFilePath: string
    	readonly configRef: EnvConfig
    }

    LoginOptions

    All fields are optional; values are read from environment variables if not provided.

    Option Type Default Description
    baseUrl string BASE_URL env Override target server base URL.
    email string USER_EMAIL env Override user email.
    password string USER_PASSWORD env Override user password.
    cacheDir string ~/.cache/libts-csrfx-auth Custom directory for session cache.
    maxRetries number 3 Maximum retry attempts.
    retryDelayMs number 100 Initial retry delay (exponential backoff).
    timeoutMs number 15000 Request timeout in milliseconds.
    sendReferer boolean true Send Referer header.
    sendOrigin boolean true Send Origin header.

    Example:

    const auth = new AuthClient({
    	baseUrl: 'https://staging.example.com',
    	email: 'test@example.com',
    	password: 'secret',
    	maxRetries: 5,
    	timeoutMs: 30000,
    })
    const session = await auth.login()

    LogoutClient

    Terminates an authenticated session.

    class LogoutClient {
    	constructor(options?: LogoutOptions)
    	logout(): Promise<void>
    	logoutWithToken(csrfToken: string): Promise<void>
    	clearCache(): Promise<void>
    	loadCache(): Promise<SessionData | null>
    	readonly configRef: EnvConfig
    }

    LogoutOptions

    Option Type Default Description
    baseUrl string BASE_URL env Override base URL.
    cacheDir string ~/.cache/libts-csrfx-auth Custom cache directory.
    timeoutMs number 15000 Request timeout.
    sendReferer boolean true Send Referer header.
    sendOrigin boolean true Send Origin header.

    Example:

    const logout = new LogoutClient({ baseUrl: 'https://staging.example.com' })
    await logout.logout()

    EnvConfig & Configuration

    Validated configuration class for endpoints and credentials.

    class EnvConfig {
    	constructor(options: EnvConfigOptions)
    	static fromEnv(overrides?: Partial<EnvConfigOptions>): EnvConfig
    	get fullLoginUrl(): string
    	get fullLogoutUrl(): string
    	hasValidCredentials(): boolean
    	readonly baseUrl: string
    	readonly loginPath: string
    	readonly logoutPath: string
    	readonly email: string
    	readonly password: string
    }

    Example:

    import { EnvConfig, loadEnv } from 'libts-csrfx-auth'

    const env = loadEnv()
    const config = new EnvConfig({
    	baseUrl: env.BASE_URL,
    	loginPath: env.LOGIN_PATH,
    	email: env.USER_EMAIL,
    	password: env.USER_PASSWORD,
    })
    console.log(config.fullLoginUrl) // https://example.com/login

    CacheManager

    Persistent session storage on disk.

    class CacheManager {
    	constructor(customDir?: string)
    	save(session: SessionData): Promise<void>
    	load(): Promise<SessionData | null>
    	clear(): Promise<void>
    	updateCsrfToken(newToken: string): Promise<void>
    	loadFresh(maxAgeMs: number): Promise<SessionData | null>
    	readonly cacheFilePath: string
    	readonly cacheDirPath: string
    }

    Utility Functions

    The library also exports several low‑level helpers:

    Function Description
    extractCsrfToken(html) Extract CSRF token from HTML (supports <input name="_token"> and <meta name="csrf-token">).
    parseCookies(headers) Parse Set-Cookie headers into a Map<string, string>.
    buildCookieHeader(cookies) Serialize a cookie map into a Cookie header string.
    parseSetCookie(cookieStr) Parse a raw Set-Cookie header into {name, value}.
    fetchWithRetry(url, options?) HTTP client with automatic retries, timeout, and exponential backoff.
    isSessionFresh(session, maxAgeMs) Check if a session is still within its freshness window.
    sessionWithCsrfToken(session, newToken) Create a new session object with updated token and timestamp.

    Error Handling

    All errors thrown by the library are instances of AuthError and include:

    • code – one of the predefined AuthErrorCode values
    • context – optional contextual information
    • timestamp – milliseconds since epoch
    • cause – original error (if any)
    • toJSON() – serialisable for logging
    • getFormattedMessage() – human‑readable string
    import { AuthError, isRetryableError, isAuthenticationError } from 'libts-csrfx-auth'

    try {
    	await auth.login()
    } catch (err) {
    	if (err instanceof AuthError) {
    		console.error(`[${err.code}] ${err.message}`)
    		if (isRetryableError(err)) {
    			console.log('Will be retried automatically')
    		}
    		if (isAuthenticationError(err)) {
    			// redirect to login page
    		}
    	}
    }

    Error Types

    Code Retryable Description
    INVALID_CREDENTIALS false Email or password missing/invalid.
    CSRF_NOT_FOUND true CSRF token not found in HTML.
    CSRF_FETCH_FAILED true Failed to fetch login page.
    CSRF_EXPIRED false Server returned HTTP 419.
    LOGIN_FAILED false Login request returned non‑2xx status.
    LOGOUT_FAILED false Logout request failed.
    NOT_AUTHENTICATED false No valid session in cache.
    NETWORK_ERROR true DNS, TLS, or socket error.
    TIMEOUT true Request timeout exceeded.
    CACHE_ERROR false File system cache read/write failure.
    VALIDATION_ERROR false HTTP 400/422 – validation error.
    TOO_MANY_REQUESTS true HTTP 429 – rate limited.
    SERVER_ERROR true HTTP 5xx – server error.
    UNAUTHORIZED false HTTP 401 – unauthorized.
    FORBIDDEN false HTTP 403 – forbidden.
    NOT_FOUND false HTTP 404 – endpoint not found.
    UNKNOWN false Catch‑all for unexpected errors.

    Retryable Errors

    The library automatically retries requests for errors where retryable === true (network, timeout, server errors, rate limits). You can also use isRetryableError(error) to check manually.

    Authentication Errors

    Use isAuthenticationError(error) to detect errors that indicate the user is not authenticated (invalid credentials, expired CSRF, 401/403, etc.).

    Environment Variables

    Create a .env file in your project root (or set system environment variables):

    Variable Required Default Description
    BASE_URL [x] – Base URL of the target server (http/https).
    LOGIN_PATH [ ] login Login endpoint path (relative).
    LOGOUT_PATH [ ] logout Logout endpoint path (relative).
    USER_EMAIL [ ] – User email address (required by AuthClient).
    USER_PASSWORD [ ] – User password (required by AuthClient).

    Variables are loaded via loadEnv(), which validates URLs, email format, and path restrictions.
